     Context

     The spec at .claude/specs/fiori_dev_spec.md describes a SAPUI5 freestyle
     app ("TravelHub")
     on top of the existing CAP services. During exploration I found the spec's
     data
     assumptions do not match the real backend, so the design adapts the spec to
      the
     actual services rather than implementing it verbatim.

     Confirmed via srv/*.cds, handlers, .http test files and seed CSVs:

     - AuthService /odata/v4/auth вҖ” POST login {loginName,password}, POST 
     refresh {refreshToken},
     GET me (Bearer). Returns {access,refresh,expiresIn} /
     {id,loginName,firstName,lastName,roles[]}.
     - CatalogService /odata/v4/catalog вҖ” Travellers, TravelledLocations,
     Destinations,

     Context

     The spec at .claude/specs/fiori_dev_spec.md describes a SAPUI5 freestyle
     app ("TravelHub")
     on top of the existing CAP services. During exploration I found the spec's
     data
     assumptions do not match the real backend, so the design adapts the spec to
      the
     actual services rather than implementing it verbatim.

     Confirmed via srv/*.cds, handlers, .http test files and seed CSVs:

     - AuthService /odata/v4/auth вҖ” POST login {loginName,password}, POST 
     refresh {refreshToken},
     GET me (Bearer). Returns {access,refresh,expiresIn} /
     {id,loginName,firstName,lastName,roles[]}.
     - CatalogService /odata/v4/catalog вҖ” Travellers, TravelledLocations,
     Destinations,
     TravellerStatus, AddressTypes, TravellerTypes. Row-level @restrict:
     traveller sees
     only own rows (userID = $user.email); ADMIN sees all.
     - UserManagement /odata/v4/user-management (ADMIN only) вҖ” Users, Roles,
     UserRoles,
     functions lockUser/unlockUser(userId), actions
     createUser/resetPassword/assignRole/unassignRole.
     - Auth identity lives in Users+UserCredentials+UserRoles, not on
     Travellers.
     Travellers is the business profile linked by userID = email = 
     Users.loginName.
     - TravelledLocations has no amount/spend field. No Addresses entity exists.
     - Seed logins (all password Welcome1!): admins superadmin@anubhav.com,
     anubhav@anubhavtrainings.com; travellers e.g. raj.sharma@example.com.
